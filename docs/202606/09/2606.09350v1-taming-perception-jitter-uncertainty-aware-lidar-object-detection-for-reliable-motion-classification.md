## 摘要
可靠的运动分类对于自动驾驶至关重要，因为对静态物体的虚假动态预测可能级联引发不必要的规划器干预。不稳定的边界框预测会导致跟踪中的速度估计出现虚假值以及错误的轨迹预测。我们提出一种易于部署的缓解策略，该策略通过异方差不确定性估计增强3D目标检测器，并在短观察窗口上应用双样本z检验来区分真实运动与抖动。该方法以最小改动集成到Autoware中，重用现有数据关联以实现极低计算开销。实验结果表明，在nuScenes上与速度阈值方法性能相当，但在真实测试中显著减少了虚假动态预测和不必要停车，原因在于记录数据中存在速度规则误判的中间抖动带。这表明不确定性感知检测与轻量级统计检验可以在噪声更大的真实环境中为自动驾驶带来实际性能提升。

## Abstract
Reliable motion classification is critical for autonomous driving, as false dynamic predictions of static objects can cascade into unnecessary planner interventions. Unstable bounding box predictions can lead to spurious velocity estimates in tracking and falsely predicted trajectories. We present a deployment-friendly mitigation strategy that augments a 3D object detector with aleatoric uncertainty estimates and applies a two-sample z-test over short observation windows to separate true motion from jitter. Integrated into Autoware with minimal changes, the approach reuses existing data association for minimal compute overhead. Empirical results show parity with velocity thresholding on nuScenes, but substantially fewer false dynamic predictions and unnecessary stops in real-world test drives, explained by the presence of an intermediate jitter band in the recorded data that speed-only rules misclassify. This demonstrates that uncertainty-aware detection and lightweight statistical testing can deliver practical performance gains for autonomous driving in noisier real-world settings.

# 2. 论文提出的方法论

## 核心思想
- 在CenterPoint 3D目标检测器上增加异方差不确定性估计头，为每个检测输出的位置（x, y）、速度（vx, vy）和偏航角（γ_yaw）预测方差（对数似然损失）。训练采用高斯负对数似然损失。
- 利用检测器提供的逐帧位置方差，对每个跟踪对象构建一个短时间观察窗口（例如10帧），将窗口分为前后两个样本组，采用两样本z检验比较两组的位置均值是否在统计上有显著差异。若z分数超过阈值α，则判定为动态；否则为静态。
  - 公式：z = |μ̄₁ - μ̄₂| / √(σ̂₁²/n₁ + σ̂₂²/n₂)，其中σ̂²为检测器预测的方差（通过观测方差均值计算），n为有效样本量（考虑到帧间相关性，将每个窗口合并为一个簇，设置n=1）。
- 对于分类为静态的物体，将其速度强制设为零，位置取窗口平均，然后输出给轨迹预测模块。

## 关键技术细节
- **不确定性估计**：在CenterPoint回归头中额外添加方差输出（log-variance），训练时使用Gaussian NLL损失；只对x, y, vx, vy, yaw建模，不建模宽、长、高、z。
- **运动分类器**：利用跟踪器的数据关联结果，避免重复计算；对每个轨道维护滑动窗口；分别对x和y方向计算z分数，取最大值与阈值比较。
- **窗口长度**：最小为2帧，默认为10帧滑动窗口；在窗口初期允许快速决策，但稳定后更可靠。
- **阈值调整**：理论阈值为1.96（对应5%显著性），但因独立性假设不完全满足，需要人工调整。
- **集成到Autoware**：仅需替换检测器为不确定性版本；通过ROS2消息中的协方差字段传递方差数据；复用跟踪器的数据关联；对静态物体置零速度和平滑位置。

1. **不确定性感知检测器**：在小样本领域适应中，不确定性头可提升mAP（从0.375提高到0.478），尤其对稀有类别改善明显。

3. **真实路测提升**：z检验方法显著减少由抖动引起的虚假动态预测和不必要停车。原因分析发现nuScenes数据中静态/动态速度云团分离良好，而真实数据中存在“抖动带”（中等速度但低z分数），速度阈值误判而z检验正确抑制。
4. **校准问题**：自有数据上检测器的方差存在低估（纵向高估、横向低估），导致z检验理论假设不完全满足，但实际表现仍然有效。
5. **不存在严重的第二类错误**（动态被误判为静态）：动态物体通常遮挡少、不确定性低，z分数高，很少误分类。

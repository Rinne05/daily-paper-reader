## 摘要
在执行延迟与结果效用之间进行权衡（即任意时刻计算）以适应动态操作需求已被证明可以增强信息物理系统的性能。本文致力于为处理激光雷达点云以进行3D目标检测的深度神经网络（DNN）实现任意时刻计算。我们提出了一种新颖方法，使处理点云为支柱或体素的模型能够进行多分辨率推理，从而允许输入根据时序要求动态缩放并在所需分辨率下处理。重要的是，我们的内存高效方法仅需部署单个DNN模型，避免了部署多个模型（每个模型针对不同输入分辨率训练）的需要。我们还引入了一种考虑截止时间的调度器，通过准确预测运行时所有可能分辨率的执行时间（由于激光雷达点云的不规则性，这具有挑战性），为任何给定输入选择最高可能分辨率。在nuScenes自动驾驶数据集上的实验结果表明，我们的方法显著优于现有的激光雷达目标检测任意时刻计算方法。最后，我们将该方法部署在模拟自动驾驶系统中，在该系统中，它始终能够在避免环境复杂性引起的非必要停滞的同时，实现无碰撞导航。

## Abstract
Making tradeoffs between execution latency and result utility (i.e., anytime computing) for adapting to dynamic operational requirements has been shown to enhance the performance of cyber-physical systems. In this work, we focus on enabling anytime computing for deep neural networks (DNNs) that process LiDAR point clouds for 3D object detection. We propose a novel method that enables multi-resolution inference for models that process point clouds as pillars or voxels, allowing the input to be dynamically scaled and processed at the resolution needed to meet timing requirements. Importantly, our memory-efficient approach requires the deployment of only a single DNN model, avoiding the need to deploy multiple models, each trained for a different input resolution. We also introduce a deadline-aware scheduler that selects the highest possible resolution for any given input by accurately predicting the execution time for all possible resolutions at runtime, which is challenging due to the irregularity of LiDAR point clouds. Experimental results on the nuScenes autonomous driving dataset demonstrate that our method significantly outperforms existing anytime computing approaches for LiDAR object detection. Finally, we deploy our approach in a simulated autonomous driving system, where it consistently enables collision-free navigation while avoiding unnecessary stalls caused by environmental complexity.

## 1. 核心问题与整体含义（研究动机和背景）
- **问题**：自动驾驶系统中的 LiDAR 3D 目标检测深度神经网络（DNN）计算开销大，难以在资源受限的嵌入式平台上同时满足高精度和低延迟要求。实际运行环境动态变化（如复杂城市低速场景 vs. 高速公路高速场景），需要算法能在精度与延迟之间灵活权衡，即实现**任意时刻计算**（anytime computing）。
- **背景**：现有任意时刻方法多聚焦于图像分类/检测或 LiDAR 的早期退出/输入切片策略，但**输入分辨率**（pillar/voxel 大小）作为关键缩放维度尚未被充分探索。简单部署多个不同分辨率模型会带来线性增加的内存开销，不实用；而单一模型在不同分辨率下评估会导致严重精度下降（如表1所示）。
- **目标**：提出一种内存高效、仅需单个 DNN 模型、支持动态分辨率缩放且能实时满足动态截止时间要求的 LiDAR 目标检测框架。

## 2. 方法论：核心思想、关键技术细节、流程
### 核心思想
- **多分辨率训练与推理**：训练时对同一批点云使用多种 pillar/voxel 尺寸分别进行前向传播，损失叠加后反向传播更新共享权重。关键创新：为每种分辨率设置**独立的批归一化（BN）层**（Resolution-Aware BN），而其余卷积层权重共享，从而解决不同分辨率下统计分布差异导致的精度损失。
- **后训练分辨率合成**（仅适用于 pillar 模型）：对每个 BN 层的每个通道，使用二次回归模型拟合训练时已知分辨率与 BN 参数的关系，从而推断未知分辨率的 BN 参数，扩展可用分辨率集合，提供更细粒度的延迟-精度权衡。
- **动态截止时间感知调度器**：
  - 分解总延迟 \(L = L_{PFE} + L_{SC} + L_{DC} + L_{PP}\)（支柱特征编码、稀疏 CNN、密集 CNN、后处理）。
  - 对 \(L_{PFE}\) 使用每个分辨率下的二次回归（基于点数量）预测；对 \(L_{DC}, L_{PP}\) 使用 99 百分位静态值；对不稳定的 \(L_{SC}\)（pillar 模型）采用**最大池化模拟稀疏卷积**的方法，预先计算各稀疏层活跃 pillar 数，再通过每层二次回归累加预测时间；对 voxel 模型则使用**离线查找表 + 历史帧匹配**。
  - 调度算法（Algorithm 1）：按分辨率从高到低扫描，返回第一个预测时间≤截止时间的分辨率；若都不满足则返回最低分辨率（最大 pillar 尺寸）。
- **其他优化**：
  - 对密集 CNN 进行**边缘裁剪**和**检测头区域限制**，减少不必要的计算。
  - **预测机制**：基于历史检测结果和速度估计，预测当前帧中可能被漏检的目标，提升召回率。
  - **区域丢弃**（仅 voxel 模型）：当预测欠时效可能导致超时，从 BEV 张量左右侧均匀裁剪部分区域以降低延迟，确保截止时间满足。

## 6. 主要结论与发现
- **MURAL 在开环评估中**：在所有截止时间下均优于固定分辨率基线和 VALO，尤其在高截止时间下显著领先；通过动态分辨率缩放，可充分利用可用时间提升精度。

- **时间预测更准**：对于 pillar 模型，最大池化模拟方法相比 VALO 的历史法误差更小，CDF 更集中。
- **闭环保验证**：

- **结论**：MURAL 是第一个实现 LiDAR 目标检测运行时分辨率动态缩放的方法，达到 SOTA 任意时刻性能，并在真实闭环模拟中提升安全性和效率。

## 7. 优点
- **内存高效**：单模型支持多分辨率，BN 层独立但参数极少，避免多模型部署。
- **通用性**：同时适用于 pillar 和 voxel 架构（CenterPoint 仅不支持后训练合成，其余均适用）。
- **后训练合成**（仅 pillar）：无需重新训练即可扩展分辨率，提供更细粒度权衡。
- **调度器预测精确**：对 pillar 模型的最大池化模拟方法新颖且准确性高，降低超时风险。
- **闭环保评估**：超过多数相关工作的开放环评估，直接验证对实际自动驾驶系统的影响（安全性、效率）。
